# T3000汉化 + AgentBridge 编译指南

## 环境准备

### 必需软件

| 软件 | 版本 | 下载 |
|------|------|------|
| Visual Studio 2019 | Community（免费） | https://visualstudio.microsoft.com/downloads/ |
| 工作负载 | 桌面开发 C++ | 安装时勾选 |
| 单个组件 | C++ MFC for latest v142 build tools | 安装时勾选 |
| Inno Setup | 6.x（免费） | https://jrsoftware.org/isdl.php |

### 安装 VS2019 步骤

1. 下载 VS2019 Community 安装程序
2. 运行安装程序，选择工作负载：
   - ☑ **桌面开发 C++**
   - ☑ **C++ MFC for latest v142 build tools (x86 & x64)**
3. 完成安装

## 编译步骤

### 方式一：一键编译（推荐）

```cmd
REM 打开 VS2019 开发者命令提示符
REM 开始菜单 → Visual Studio 2019 → x86 Native Tools Command Prompt

cd /d D:\T3000_Building_Automation_System

REM 一键编译
build_zh.bat
```

### 方式二：手动编译

#### 1. 打开解决方案

```
文件 → 打开 → 项目/解决方案
选择: T3000 - VS2019.sln
```

#### 2. 配置构建选项

```
解决方案配置: Release
解决方案平台: Win32
```

#### 3. 编译主程序

```
生成 → 生成解决方案 (Ctrl+Shift+B)
```

#### 4. 编译 C# 控件库

```
文件 → 打开 → 项目/解决方案
选择: T3000Controls\T3000Controls.sln
配置: Release | Any CPU
生成 → 生成解决方案
```

#### 5. 编译安装包

```
右键 T3000_zh.iss → Compile
或运行: build_installer.bat
```

## AgentBridge 配置

### 通过 MFC 对话框配置

1. 工具 → AgentBridge（需要在菜单中添加）
2. 设置端口号（默认 8080）
3. 设置 API Key（可选）
4. 勾选"启用 AgentBridge"
5. 点击"启动"

### 通过代码启动

在 `T3000.cpp` 的 `InitInstance()` 中已添加：

```cpp
// ★ 中文汉化：初始化语言包
m_locale.AddCatalogLookupPath(g_strExePth);
m_locale.SetLanguage(CLanguageLocale::LANGUAGE_CHINESE);
gSetMenuStrings(m_pMainWnd->m_hWnd);

// ★ AgentBridge 启动
g_AgentBridge.SetPort(8080);
g_AgentBridge.SetApiKey(_T("your-api-key-here"));
g_AgentBridge.SetEnabled(true);
g_AgentBridge.SetMainFrame(this);
g_AgentBridge.Initialize();
g_AgentBridge.Start();
```

## 常见问题

### 编译错误：找不到 MFC

```
错误: 无法打开包括文件: 'afxwin.h'
解决: 安装 C++ MFC 组件
     VS2019 安装程序 → 修改 → 单个组件 → 搜索 "MFC" → 勾选安装
```

### 编译错误：HTTP API

```
错误: 无法解析的外部符号 _HttpInitialize@8
解决: 确保链接 httpapi.lib
     项目属性 → 链接器 → 输入 → 附加依赖项 → 添加 httpapi.lib
```

### 编译错误：WebSocket

```
错误: 找不到 openssl/sha.h
解决: 安装 OpenSSL 或使用简化版 WebSocket 实现
     或从 https://slproweb.com/products/Win32OpenSSL.html 下载
```

### 端口被占用

```
错误: HttpRegisterUrl failed
解决: 更换端口号
     工具 → AgentBridge → 修改端口号
     或: netstat -ano | findstr :8080 查看占用进程
```

### AgentBridge 未启动

```
检查:
1. 注册表是否有语言设置
2. MainFrm 指针是否正确设置
3. 查看调试输出: [AgentBridge] 日志
```
